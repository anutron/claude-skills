## Working Directory

**For ephemeral scratch files** (investigations, one-off scripts, data exports), use `working/YYYY-MM-DD/`:

- Create `working/<today's date>/` when you need to write throwaway files
- Subfolders can include a descriptive suffix: `working/2025-02-15-blaze-email-investigation/`
- Files persist across sessions for reference but aren't tracked in git
- Don't clean up old folders — they serve as a searchable history
